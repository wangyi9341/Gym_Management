# 力量健身管理系统（WPF）测试计划（简版）

## 1. 目标
- 验证 `dist\力量健身管理系统.exe` 在大数据量（约 1000 条主数据）下的稳定性、关键业务流程可用性、主要边界条件与异常提示。
- 覆盖：教练管理、私教课会员（缴费/消课）、年卡会员（到期/续费/筛选）、首页 Dashboard（提醒与聚合）。

## 2. 范围
### 2.1 功能范围（In Scope）
- 启动：首次运行自动建表、读取/生成配置、错误提示与日志落盘
- 教练：CRUD、搜索、重复工号拦截
- 私教课会员：CRUD、缴费记录、消课记录、剩余课时校验、明细刷新、级联删除
- 年卡会员：CRUD、筛选（正常/即将到期/已过期）、续费规则、到期提醒逻辑
- Dashboard：聚合计数、到期提醒横幅、低剩余课时名单、定时刷新（人工观察）

### 2.2 非范围（Out of Scope）
- 会员端 App（项目不包含）
- 多人并发/多机网络共享的 SQL Server 压测（可后续补充）
- 安装包（Inno Setup）兼容性全量验证（可后续补充）

## 3. 测试类型与方法
- 自动化脚本（数据准备 + 基础校验）：`qa\GymManager.QaRunner`
- 人工回归（UI/交互体验）：按报告中的“人工 UI 回归建议”逐项勾选

## 4. 测试数据（目标分布）
- Coaches：100
- 私教会员：500（含 0 课时、低剩余、较大课时、特殊字符姓名/电话等）
- 年卡会员：400（含已过期、即将到期、正常分布；含边界日期 start=end）
- 关联数据：私教会员含 0~3 条缴费记录、0~5 条消课记录

## 5. 退出准则（Exit Criteria）
- 关键用例全部通过：启动、CRUD、消课/续费规则、筛选/提醒无逻辑错误
- 大数据量下无明显卡死/崩溃；日志无未处理异常
- 若存在缺陷：已记录问题、可复现步骤、影响范围与建议修复方案
